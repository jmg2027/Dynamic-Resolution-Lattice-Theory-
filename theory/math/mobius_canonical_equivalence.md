# Möbius Canonical Equivalence on Cuts

**Status**: Closed across 5 distinct equality domains (11 files,
90 PURE / 0 DIRTY).

## Overview

The Möbius matrix `P = [[2,1],[1,1]]` — the algebraic
representation of 213's atomic signature `(NS, NT, d) = (3, 2, 5)`
with `(trace, det, disc) = (NS, NS−NT, NS+NT)` — carries TWO
structural readings on 213 cuts:

  (a) An **orbit structure**: mediant-closure of the two seeds
      `(0, 1)` and `(1, 0)` under the rule `(a, b) ⊕ (c, d) :=
      (a+c, b+d)` enumerates every coprime pair in ℕ × ℕ (the
      Stern-Brocot tree).  Agreement of cuts on this orbit is
      the canonical equivalence relation, equivalent to pointwise
      `cutEq` up to a single (0, 0) boundary condition that is
      automatic on every `constCut`-built cut.

  (b) A **symplectic invariant**: the cross-product
      `m · k' − m' · k = ±1` between adjacent mediant-tree pairs
      is the unit of the quadratic ring whose discriminant is
      `disc(P) = 5 = NS + NT`.  On the P-orbit `Pseq seedZero`
      this realises as the Pell-Fibonacci identity
      `a² + 1 = a·b + b²`.

The atomic signature itself appears at orbit depth 2:
`Pseq seedZero 2 = (NS, NT) = (3, 2)`.  Hence the SAME matrix
that algebraically encodes atomicity ALSO defines the canonical
equivalence on cuts AND carries the unit invariant — three
readings sharing one algebraic object.

## The narrative

### The conjecture and its closure

The framework carries multiple equality definitions across its
subsystems:

  · `cutEq` — pointwise `Nat → Nat → Bool` equality (Real213)
  · `ValidCutN.is_at_denom` — bundled witness in `ValidCutN N`
  · `signedEq` (= `∀ m k, signedEqAt`) — cross-additive equality
    on signed cuts
  · `Adjacent` on dyadic brackets, `LensMap` (categorical), etc.

The conjecture: each of these is a projection of one canonical
equivalence relation, generated by the Möbius matrix `P` acting
on `ℕ × ℕ`.

The closure realises this as a chain of strictly weakening
relations on `Nat → Nat → Bool`:

```
cutEq cx cy   ⇔   sternBrocotEq cx cy ∧ cx 0 0 = cy 0 0
             →   sternBrocotEq cx cy   (drops (0, 0) constraint)
             →   mobiusEq cx cy        (restricts to two P-orbits)
```

Forward implications all unconditional.  The first is bidirectional
because `SternBrocotReachable` covers every `(m, k)` with
`m + k ≥ 1` (`reachable_of_pos`), leaving only `(0, 0)` as a
boundary outside the mediant closure.  For canonical 213 cuts the
(0, 0) condition is automatic (`constCut a N 0 0 = true` always).

### The P-orbit and the Stern-Brocot mediant

The matrix `P = [[2,1],[1,1]]` acts on `(m, k) ↦ (2m+k, m+k)`.
From the two boundary seeds `seedZero = (0, 1)` and
`seedInf = (1, 0)`, the P-iteration generates:

```
Pseq seedZero n  :  (0,1), (1,1), (3,2), (8,5), (21,13), (55,34), ...
Pseq seedInf  n  :  (1,0), (2,1), (5,3), (13,8), (34,21), (89,55), ...
```

These coincide (shifted) with `P_numerator` / `P_denominator`
from `Lib/Math/Mobius213.lean` — Fibonacci-even/odd Pell
convergents whose ratios approach `φ²` and `1/φ²`.

**Pure P-iteration is one diagonal per seed.**  P factors as
`R · L` in the standard SL₂(ℤ) generators
`L = [[1,0],[1,1]]`, `R = [[1,1],[0,1]]`; the Stern-Brocot tree
uses L and R as *separate* operations.  So P-orbit equivalence
(`mobiusEq`) is genuinely weaker than full Stern-Brocot
equivalence (`sternBrocotEq`).

The mediant closure recovers the tree.  `SternBrocotReachable`
is defined inductively with constructors `seedZero`, `seedInf`,
and `mediant (a, b) ⊕ (c, d) = (a+c, b+d)`.  Two simple
"extension" lemmas

```
reachable_succ_fst : SR (m, k) → SR (m + 1, k)   -- ⊕ seedInf
reachable_succ_snd : SR (m, k) → SR (m, k + 1)   -- ⊕ seedZero
```

give full ℕ × ℕ coverage modulo `(0, 0)` (`reachable_of_pos`).
Combined with the Pell-orbit identities `Pseq_seedInf_components`
(the cross-orbit relation `(Pseq seedInf n) =
((Pseq seedZero n).1 + .2, (Pseq seedZero n).1)`), one proves
`Pseq_reachable` — both P-orbits embed into the Stern-Brocot
tree via the mediant identities

```
Pseq seedZero (n+1) = mediant(Pseq seedZero n, Pseq seedInf n)
Pseq seedInf  (n+1) = mediant(Pseq seedZero (n+1), Pseq seedInf n)
```

### The (0, 0) boundary condition

`SternBrocotReachable` cannot reach `(0, 0)` — every constructor
preserves the property `m + k ≥ 1`.  But arbitrary `Nat → Nat →
Bool` functions can disagree at `(0, 0)` without affecting
`sternBrocotEq`.  The general bidirectional bridge is therefore

```
cutEq cx cy  ⇔  sternBrocotEq cx cy  ∧  cx 0 0 = cy 0 0
```

On every cut built from `constCut a N` (where
`constCut a N 0 0 = decide (a · 0 ≤ N · 0) = decide (0 ≤ 0) =
true`) the condition is automatic.  This includes:

  · `ValidCutN N` (any bundled `cut` field, via `is_at_denom`
    transport — `validCutN_zero_zero`)
  · `cutSumN N`, `cutMul`, and their iterates
  · The cross-sum cuts `cutSum (pos s) (neg t)` underlying
    `signedEq`

The bridge therefore simplifies to a direct `cutEq ↔
sternBrocotEq` for every canonical 213 cut representation.

### Algebra preservation

`ValidCutN N`'s bundled addition `addN` lifts to a bilinear
Stern-Brocot congruence (`addN_sternBrocotEq`):

```
sternBrocotEq vx.cut vx'.cut
  → sternBrocotEq vy.cut vy'.cut
  → sternBrocotEq (addN N hN vx vy).cut (addN N hN vx' vy').cut
```

Every Wave 13 theorem on `ValidCutN N` (`cutSumN_assoc_valid`,
`cutSumN_comm_valid`, `nvalidcut_all_naturals_capstone`, plus the
per-N instances `fifth_assoc_1_2_1`, `seventh_assoc_2_3_5`,
`eleventh_assoc_1_4_6`) lifts to Stern-Brocot-orbit form by
composing with this congruence.

### The Pell unit invariant

The symplectic content of `det(P) = 1` lifts to consecutive
P-orbit pairs.  On the seedZero orbit, the Nat identity

```
(Pseq seedZero n).1 · (Pseq seedZero n).1 + 1
  = (Pseq seedZero n).1 · (Pseq seedZero n).2
    + (Pseq seedZero n).2 · (Pseq seedZero n).2
```

holds for every depth `n` (`Pseq_seedZero_pell_invariant`).
Combined with the cross-orbit relation, this gives the
cross-product invariant

```
(Pseq seedZero n).1 · (Pseq seedInf n).2 + 1
  = (Pseq seedZero n).2 · (Pseq seedInf n).1
```

(`Pseq_cross_pell_invariant`) — the Pell unit value `-1` (in Int)
reading `NT − NS = 2 − 3` in 213 terms.  The discriminant
`disc(P) = 5 = NS + NT` is the algebraic "size" of the quadratic
ring `ℤ[φ²]` whose units this invariant measures.

The Nat-side identity matches the Int-side
`mobius_213_pell_unit_invariant_forall` from
`Lib/Math/Mobius213.lean` without going through coercion.

### The atomicity anchor

The P-orbit hits the atomic signature at depth 2:
`Pseq seedZero 2 = (NS, NT) = (3, 2)`
(`pseq_seedZero_realises_NS_NT`).  Symmetrically
`Pseq seedInf 2 = (NS + NT, NS) = (5, 3)`.

This connects to `Theory.Atomicity.Five.atomic_iff_five` (the
unique alive-decomposition Nat is 5) and
`mobius_213_discriminant` (`disc(P) = trace² − 4·det = 5`).

Six readings of `5` consolidate in `disc_atom_orbit_master`:

```
NS + NT = 5 = d                       (Counts / partition_sum)
∧ 3² = 4·1 + 5                        (Möbius discriminant, Nat-form)
∧ Atomic (NS + NT)                    (atomic_iff_five)
∧ (Pseq seedInf 2).1 = NS + NT        (orbit depth-2)
∧ SternBrocotReachable (NS, NT)       (mediant of (2,1), (1,1))
∧ SternBrocotReachable (NS + NT, NS)  (mediant of (3,2), (2,1))
```

The matrix `P` therefore writes the atomicity signature directly
into its second-depth Stern-Brocot orbit.  The same algebraic
object encodes:
  · the discriminant `5` (algebra),
  · the unique atomic Nat (combinatorics),
  · the canonical equivalence on cuts (orbits),
  · the symplectic cross-product invariant (det = 1),
  · the second-depth orbit position (Pseq).

## Generalisation

`Mobius213CrossDomainMeta.cross_domain_meta_unification` bundles
the five domain-bridges into one master statement.  The
structural principle: for each 213-internal equality definition,
the canonical Möbius-orbit equivalence is determined by the
*coordinate shape* of the underlying domain.

| Coordinate shape | Domain | Möbius reading |
|---|---|---|
| `Nat × Nat` | `Nat → Nat → Bool` cuts | `sternBrocotEq` (mediant closure) + (0, 0) cond |
| `Nat × Nat` | `ValidCutN N` cut fields | `sternBrocotEq` (auto (0, 0)) |
| `Nat × Nat` (cross-sum) | `SignedCut` | `sternBrocotEq` on `cutSum (pos s) (neg t)` |
| `Nat` (projected) | `ZpSeq p` | `ZpMobiusPairEq` via SB pair coverage |
| Function-level | `DyadicBracket` walls | `sternBrocotEq` via `adjacent_walls_match` |

Every framework-internal equality definition factors through the
appropriate Möbius-orbit reading on its coordinate shape.
