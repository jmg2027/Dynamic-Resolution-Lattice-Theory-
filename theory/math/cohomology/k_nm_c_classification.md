# K_{NS,NT}^{(c)} classification: c-counter, Stern-Brocot path, K_{3,3} / K_{4,3} extensions

Math-side companion to `bipartite.md` and `k32_higher_cohomology.md`:
the universal classification of bipartite multigraphs
`K_{NS, NT}^{(c)}` along three orthogonal axes — Stern-Brocot
position, gcd-scale, and multiplicity-layer count — and the
parametric `codim ≥ c` cohomology theorem that holds at every
position.

K_{3,2}^{(c=2)}-local higher cohomology (Steenrod algebra, cup_i
ladder, Steenrod-Whitehead bridge) is hosted in
`k32_higher_cohomology.md`; this chapter generalises the
multiplicity / graph-shape axes outward to the full
`K_{NS, NT}^{(c)}` family.

## Three orthogonal axes

Every bipartite multigraph `K_{NS, NT}^{(c)}` carries a unique
position in a 3-axis lattice:

  · **Stern-Brocot path** `(NS / gcd(NS,NT), NT / gcd(NS,NT))` —
    coprime reduction.  Every coprime pair appears uniquely in
    the Stern-Brocot tree generated by mediants of the seeds
    `(0, 1), (1, 0)`.
  · **Scale factor** `gcd(NS, NT)` — the multiplicative scaling
    along the diagonal `{(N, N) : N ∈ Nat}`.  Coprime pairs have
    gcd = 1; symmetric pairs have gcd = NS = NT.
  · **Multiplicity layer count** `c` — the number of independent
    edge layers, orthogonal to the (NS, NT) graph shape.

The first two axes determine the graph topology (vertex / edge
count, simple-cycle face structure); the third axis controls the
enriched 2-complex's H² dimension via the c-counter.

## Concrete instances

| Graph | (NS, NT) | Stern-Brocot position | Pell unit |
|---|---|---|---|
| `K_{3,2}^{(c)}` | (3, 2) | `Pseq seedZero 2` orbit point (depth 2) | ✓ `3² + 1 = 3·2 + 2²` |
| `K_{4,3}^{(c)}` | (4, 3) | mediant of (1, 1) and (3, 2) — tree interior | ✗ off both orbits |
| `K_{5,3}^{(c)}` | (5, 3) | `Pseq seedInf 2` orbit point | (sign-flipped Pell unit) |
| `K_{3,3}^{(c)}` | (3, 3) = 3·(1, 1) | scale-3 of Stern-Brocot root | ✗ off seedZero |

The two thin chains inside the Stern-Brocot tree:

  · `Pseq seedZero`: (0,1), (1,1), (3,2), (8,5), (21,13), …
  · `Pseq seedInf`:  (1,0), (2,1), (5,3), (13,8), (34,21), …

Both satisfy the SL₂(ℤ) determinant unit `|ab + b² − a²| = 1`
(signed: +1 on seedZero, −1 on seedInf).  Tree-interior mediants
(like K_{4,3}) lie BETWEEN these chains.

The (3, 2) anchor coincides with the algebraic atomic signature
forced in `physics/foundations/atomic_constants.md` — the
Stern-Brocot reading shows the SAME pair as the depth-2 image of
`(0, 1)` under Möbius P-iteration.

## c-counter as multiplicity-layer count

The original conjecture "cup-image codim = `c − 1` at Massey
depth `c + 2`" placed `c`-dependence in topological invariants
of the simple-cycle complex.  This is falsified at every
multiplicity: in the simple complex codim stays at 1 invariant
of `c` (proved at c = 2 via `V33` and at c = 3 via
`V33c3.c3_conjecture_refined`).

The correct location: the **enriched 2-complex** that admits
multi-multiplicity face cycles.

### The enriched 2-complex (categorical direct sum of c copies)

The c-multiplicity layers in `K_{NS, NT}^{(c)}` have DISJOINT
edge sets:

```
edges at mult m = { c·(NT·i + j) + m : i ∈ Fin NS, j ∈ Fin NT }
                = { c·k + m : k ∈ Fin (NS·NT) }
```

For distinct `m, m' ∈ Fin c`, these sets don't overlap.  Hence

```
C¹_enr = ⊕_{m ∈ Fin c} C¹_simple_m
C²_enr = ⊕_{m ∈ Fin c} C²_simple_m
δ¹_enr = ⊕_m δ¹_simple_m
ψ_enr  = ⊕_m ψ_simple_m
```

The enriched complex is a CATEGORICAL DIRECT SUM of `c` simple
complexes — cohomology distributes:

```
H²_enr           = ⊕_m H²_simple_m
cup-image_enr    = ⊕_m cup-image_simple_m
codim_enr        = c · (per-layer codim)
```

For K_{3,3}^{(c=2)}: per-layer codim = 1, total codim = 2.  For
c = 3: 3.  General: `codim = c · 1 = c`.

`theory/essays/disjoint_layers_as_direct_sum.md` exposits this
direct-sum structure as the categorical reason c becomes a
discriminator-count rather than a depth parameter.

### ψ-discriminator as universal block functional

Per multiplicity layer `m ∈ Fin c`,

```
ψ_m(v) := XOR_{f ∈ layer-m faces} v(f)
```

This functional:

  · vanishes on `imδ¹_m` — each layer-m edge appears in
    `(NS − 1)(NT − 1)` layer-m faces (even when both ≥ 2);
  · vanishes on cup-images of cocycles supported in OTHER
    layers (edge-disjoint property above);
  · is NONZERO on layer-m face indicators outside cup-image of
    same-layer cocycles.

So the `c` independent `ψ_m`-functionals lift to `c` independent
2-cocycle classes in H², each cup-trivial against its own
layer's cocycles.

### Parametric ∀c codim ≥ c

`V33EnrichedParametric.parametric_c_independent_h2_classes`
(PURE):

```
∀ (c : Nat) (m m' : Fin c),
  psi_layer c m' (e_face_layer c m) = decide (m.val = m'.val)
  ∧ ∀ σ, e_face_layer c m ≠ delta1_enr_param c σ
```

`c` Kronecker-δ ψ-signatures, all non-coboundaries.  The proof
uses an abstract face value space `Fin 3 → Fin 3 → Fin c → Bool`
to keep the argument `omega`/`simp`/`funext`/`dite`-free.

The "(c−1)-codim" form is the WRONG parametrisation; the correct
statement is `codim ≥ c` in the enriched complex.

### Bottom-layer bilateral kill

At the bottom layer (m = 0), `ψ_0` simultaneously kills the cup
image of every S-star generator AND every T-incidence generator:

```
parametric_bottom_layer_full_kill_capstone :
  ∀ (c : Nat) (hc : 0 < c),
    (∀ β, ∀ (i j : Fin 3),
       psi_layer c ⟨0, hc⟩ (cupOpp_param c (starS i) β) = false)
    ∧ (∀ α, ∀ (i j : Fin 3),
       psi_layer c ⟨0, hc⟩ (cupOpp_param c α (incidT j)) = false)
```

(`V33EnrichedParametric.lean`, PURE.)  The proof rests on a
proof-term-matching insight — instead of `cases β ⟨k, by_decide⟩`
with a val-literal index, use `cases β (edge_idx c ⟨i, _⟩ ⟨j, _⟩
⟨0, hc⟩)` with the SAME edge_idx construction that appears in
the unfolded goal.  Fin proof terms are preserved exactly,
enabling rfl-reductions to chain through.

### Arbitrary-m bilateral kill

The bottom-layer kill generalises to **every** multiplicity layer:

```
parametric_arbitrary_m_full_kill_capstone :
  ∀ (c : Nat) (m : Fin c),
    (∀ β, ∀ (i : Fin 3),
       psi_layer c m (cupOpp_param c (starS c i m) β) = false)
    ∧ (∀ α, ∀ (j : Fin 3),
       psi_layer c m (cupOpp_param c α (incidT c j m)) = false)
```

(`V33EnrichedParametric.lean` §20, PURE.)  Closure mechanism:
`starS c i m` and `incidT c j m` at a same-layer `edge_idx c i' j' m`
reduce to layer-independent triple Bool disjunctions

```
starS_at_edge_idx_same_m :
  starS c i m (edge_idx c i' j' m)
  = (3·i' + j' == 3·i) || (3·i' + j' == 3·i + 1) || (3·i' + j' == 3·i + 2)
```

by cancelling the common `9·m.val` offset via `nat_decide_add_left_assoc{1,2}`
(propext-free Nat cancellation in `Infrastructure/NatBeqHelpers.lean`).
After the offset is gone, the 6/9-edge β case-bash that closed the
bottom-layer kill closes every layer's kill identically — the same
proof tactic runs for every `m`.

The `9·m.val` cancellation is the **proof-level shadow** of the
edge-disjointness direct-sum decomposition (`disjoint_layers_as_direct_sum.md`):
every layer is a translate of layer 0 along the offset index, and
the Nat.beq cancellation absorbs that translate.  See
`theory/essays/multiplicity_layer_uniformity.md` for the
structural reading.

### Surface-form caveat for the cancellation

`e.val == k` for `e : Fin n` (with `e.val : Nat`) desugars to
`decide (e.val = k)` via the generic `[DecidableEq α] ⇒ BEq α`
instance — **not** `Nat.beq e.val k`.  The infrastructure therefore
carries **both** surface forms:

  · `nat_beq_add_left_assoc{1,2}` (Nat.beq form) — for callers
    using the function directly
  · `nat_decide_add_left_assoc{1,2}` (decide / `==` form) — for
    callers (like `starS`/`incidT`) whose unfolded goal emits `==`

The pure cancellation core is `nat_add_left_cancel_pure`
(propext-free Nat left-cancellation by structural recursion on `a`).
This avoids `Nat.add_left_cancel` (which depends on `propext`) in
the ∅-axiom chain.

### Massey witness at every multiplicity

The ψ_m discriminator is realised by a concrete 4-fold Massey
product at each layer.  Parametric η-cochains

```
eta_ab_layer m := indicator on (0, 1, m), (0, 2, m)
eta_cd_layer m := indicator on (1, 1, m)
```

give `cupOpp(η_ab^m, η_cd^m)` hitting the layer-m 5th dim:
`ψ_m(rep₄_m) = 1`, decide-verified at concrete `c ∈ {2, …, 12}`
via `psi_layer_rep4_eq_true_c{2..12}`.  Each layer has its own
Massey witness; `c` independent Massey reps span the `c` "extra"
H² directions.

## K_{3,3} ↔ Möbius P bridges

K_{3,3}^{(c=2)} differs from K_{3,2}^{(c=2)} in landing OFF the
canonical Pseq seedZero orbit (3, 3 lies on the diagonal
`{(N, N)}`, not on either thin chain).  Three bridges expose
the Möbius P structure that survives the off-orbit position.

### Vertex state class — NS-scaled depth-1

`Mobius213K33StateClass.state_class_NSscaled_pell_capstone`
(PURE): K_{3,3}^{(c=2)}'s all-true vertex cochain projects to
state class `(3, 3) = NS · Pseq seedZero 1`.  Iterating P:
`Pstep^n (3, 3) = NS · Pseq seedZero (n+1)`.  Both components
satisfy the Pell-Fibonacci recurrence `a(n+2) + a(n) = NS · a(n+1)`
on the NS-scaled orbit.

Unlike K_{3,2} (whose state lies directly on the orbit), K_{3,3}
lives on the `NS`-scaled diagonal — the symmetric subspace
invariant under the swap involution.  Under `Pstep`, the
diagonal goes off-diagonal: `Pstep(N, N) = (3N, 2N)`, hitting
the symmetric direction `(3, 2)`-style scaled by N.

### Edge multiplicity saturation at c = 3

`Mobius213K33c3StateClass.multCount_master` (PURE):
`multCount allTrueE = (9, 9, 9)`.  At c = 3 the three
multiplicity layers each carry exactly `NS · NT = 9` cross-pair
edges.  The uniform `(9, 9, 9)` signature is the edge-level
mirror of the `(3, 3)` vertex state class — the all-true edge
cochain saturates each layer's 9-slot exactly.

### Numerical signature bridge

`Mobius213K33Bridge.k33_signature`: `6 = 2 · NS`, `9 = NS · NS`
— the vertex count is `NT_d · NS` (with `NT_d = 2`, the
diagonal partition factor) and the cross-pair count is `NS²`.
Connects the cohomology-side counts to Möbius P invariants
directly.

### K_{3,3} unified master

`K33Unified.k33_unified_master` (PURE) bundles all four tracks:

  · (a) parametric ∀c codim ≥ c (`V33EnrichedParametric`)
  · (b) vertex state class `(3, 3)` (`Mobius213K33StateClass`)
  · (c) c = 3 edge mult-saturation `(9, 9, 9)`
        (`Mobius213K33c3StateClass`)
  · (d) numerical signature `6 = 2·NS`, `9 = NS²`
        (`Mobius213K33Bridge`)

A single 4-conjunct ∅-axiom proposition reading K_{3,3}^{(c)}
identically across cohomological, state-class, and numerical
frames.

## K_{4,3}^{(c=2)} — first asymmetric instance

K_{4,3} is the first asymmetric (NS ≠ NT) instance of the
classification, sitting at the Stern-Brocot mediant of (1, 1)
and (3, 2) — interior to the tree, not on either thin chain.

Base structure (`V43.lean`, PURE):

  · 7 vertices (4 + 3)
  · 24 edges (`NS · NT · c = 4 · 3 · 2`)
  · 18 simple 4-cycle faces (`NS.choose 2 · NT.choose 2 = 6 · 3`)
  · cycle space dimension 6
  · H² ≥ F₂¹² (face count minus dependence relations, lower
    bound; exact dim awaits explicit cup-image computation)

The 6 S-row dependence relations — at each S-pair `(i, j) ∈
{(0,1), (0,2), (0,3), (1,2), (1,3), (2,3)}`, the 3 T-pair
4-cycles sum to zero in F₂ — are all proved in
`V43.K43_all_S_row_deps_bundle`:

```
face_dep_S01_at_K43 ∧ face_dep_S02_at_K43 ∧ face_dep_S03_at_K43
∧ face_dep_S12_at_K43 ∧ face_dep_S13_at_K43 ∧ face_dep_S23_at_K43
```

## Cross-graph universality of the S-row pattern

`CrossGraphPattern.cross_graph_S_row_dependence_pattern` (PURE)
bundles the K_{3,3} and K_{4,3} S-row dependences as instances of
ONE universal fact: at NT = 3, the 3 T-pair 4-cycles at any
S-pair sum to zero in F₂.

NS does NOT enter the proof — only NT = 3 and the specific S-pair
structure matter.  The same proof template ports across
bipartite graphs sharing NT = 3.  K_{3,3} (NS = 3) gives 3
S-pair instances; K_{4,3} (NS = 4) gives 6.  General K_{NS, 3}
gives `NS.choose 2`.

This cross-graph reusability is the operational content of the
Stern-Brocot classification's "axis decoupling": cohomological
proofs depending only on NT = 3 transport along every NT-fibre
of the Stern-Brocot tree.

## Mediant cohomology functor (count level)

The Stern-Brocot mediant `(NS₁, NT₁) ⊕ (NS₂, NT₂) = (NS₁+NS₂,
NT₁+NT₂)` lifts to a **Vandermonde decomposition** of every
cell count.  `MediantCohomologyFunctor.lean` (22 PURE, ∅-axiom):

  · **Vandermonde-2 core**: `binom (a + b) 2 = binom a 2 +
    binom b 2 + a · b` (`binom_add_2`).  The combinatorial
    heart — S-pairs of K_{a+b, *} split into intra-a, intra-b,
    and cross-ab pairs.
  · **Vertex 2-term additive**:
    `V(a+c, b+d) = V(a, b) + V(c, d)` (`vertexCount_mediant`).
  · **Edge 4-term Vandermonde**:
    `E^m(a+c, b+d) = E^m(a, b) + E^m(a, d) + E^m(c, b) +
    E^m(c, d)` (`edgeCount_mediant`).  Four edge classes:
    inner-1, cross-12, cross-21, inner-2.
  · **Face factored Vandermonde²**:
    `F(a+c, b+d) = (binom a 2 + binom c 2 + a·c) ·
    (binom b 2 + binom d 2 + b·d)` (`faceCount_mediant_factored`).
    9 terms total, each pairing one of three S-pair sources
    (intra-a / intra-c / cross-ac) with one of three T-pair
    sources (intra-b / intra-d / cross-bd).

Concrete marquee instance, K_{4,3} = K_{1,1} ⊕ K_{3,2} at
c = 2:

  · `7 = 2 + 5` — vertex additivity
  · `24 = 2 + 4 + 6 + 12` — edge 4-term
  · `18 = (0 + 3 + 3) · (0 + 1 + 2) = 6 · 3 = 18`
    — face factored Vandermonde² (4 of 9 terms non-zero;
    `binom 1 2 = 0` zeros out the remaining 5)

Anchors:
`K43_{vertex,edge,face}_from_mediant`,
`K43_face_9term_evaluation`,
`countTriple_mediant_decomposition` (3-component algebra law),
`mediant_cohomology_functor_capstone` (7-conjunct master).

The functor structure: assigning
`(NS, NT) ↦ (vertexCount, edgeCount, faceCount)` factors
through the Stern-Brocot mediant inductive structure.
Since every coprime `(NS, NT) ≥ (1, 0)` is Stern-Brocot
reachable (`Mobius213SternBrocot.reachable_of_pos`), every
`K_{NS, NT}^{(c)}` count is computable as a finite sum of
products of `binom` values along the Stern-Brocot path from
seeds to (NS, NT).

`theory/essays/stern_brocot_as_universal_lattice.md` discusses
the structural significance.  The lift from cell-count
Vandermonde to actual cochain-space / Massey-class
decomposition remains open.
