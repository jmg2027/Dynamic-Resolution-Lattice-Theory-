# K_{NS,NT}^{(c)} classification: c-counter, Stern-Brocot path, K_{3,3} / K_{4,3} extensions

Math-side companion to `bipartite.md` and `k32_higher_cohomology.md`:
the universal classification of bipartite multigraphs
`K_{NS, NT}^{(c)}` along three orthogonal axes вҖ” Stern-Brocot
position, gcd-scale, and multiplicity-layer count вҖ” and the
parametric `codim вүҘ c` cohomology theorem that holds at every
position.

K_{3,2}^{(c=2)}-local higher cohomology (Steenrod algebra, cup_i
ladder, Steenrod-Whitehead bridge) is hosted in
`k32_higher_cohomology.md`; this chapter generalises the
multiplicity / graph-shape axes outward to the full
`K_{NS, NT}^{(c)}` family.

## Three orthogonal axes

Every bipartite multigraph `K_{NS, NT}^{(c)}` carries a unique
position in a 3-axis lattice:

  В· **Stern-Brocot path** `(NS / gcd(NS,NT), NT / gcd(NS,NT))` вҖ”
    coprime reduction.  Every coprime pair appears uniquely in
    the Stern-Brocot tree generated by mediants of the seeds
    `(0, 1), (1, 0)`.
  В· **Scale factor** `gcd(NS, NT)` вҖ” the multiplicative scaling
    along the diagonal `{(N, N) : N вҲҲ Nat}`.  Coprime pairs have
    gcd = 1; symmetric pairs have gcd = NS = NT.
  В· **Multiplicity layer count** `c` вҖ” the number of independent
    edge layers, orthogonal to the (NS, NT) graph shape.

The first two axes determine the graph topology (vertex / edge
count, simple-cycle face structure); the third axis controls the
enriched 2-complex's HВІ dimension via the c-counter.

## c-counter as multiplicity-layer count

The original conjecture "cup-image codim = `c вҲ’ 1` at Massey
depth `c + 2`" placed `c`-dependence in topological invariants
of the simple-cycle complex.  This is falsified at every
multiplicity: in the simple complex codim stays at 1 invariant
of `c` (proved at c = 2 via `V33` and at c = 3 via
`V33c3.c3_conjecture_refined`).

The correct location: the **enriched 2-complex** that admits
multi-multiplicity face cycles.

### The enriched 2-complex (categorical direct sum of c copies)

The c-multiplicity layers in `K_{NS, NT}^{(c)}` have DISJOINT
edge sets:

```
edges at mult m = { cВ·(NTВ·i + j) + m : i вҲҲ Fin NS, j вҲҲ Fin NT }
                = { cВ·k + m : k вҲҲ Fin (NSВ·NT) }
```

For distinct `m, m' вҲҲ Fin c`, these sets don't overlap.  Hence

```
CВ№_enr = вҠ•_{m вҲҲ Fin c} CВ№_simple_m
CВІ_enr = вҠ•_{m вҲҲ Fin c} CВІ_simple_m
ОҙВ№_enr = вҠ•_m ОҙВ№_simple_m
ПҲ_enr  = вҠ•_m ПҲ_simple_m
```

The enriched complex is a CATEGORICAL DIRECT SUM of `c` simple
complexes вҖ” cohomology distributes:

```
HВІ_enr           = вҠ•_m HВІ_simple_m
cup-image_enr    = вҠ•_m cup-image_simple_m
codim_enr        = c В· (per-layer codim)
```

For K_{3,3}^{(c=2)}: per-layer codim = 1, total codim = 2.  For
c = 3: 3.  General: `codim = c В· 1 = c`.

`theory/essays/disjoint_layers_as_direct_sum.md` exposits this
direct-sum structure as the categorical reason c becomes a
discriminator-count rather than a depth parameter.

### ПҲ-discriminator as universal block functional

Per multiplicity layer `m вҲҲ Fin c`,

```
ПҲ_m(v) := XOR_{f вҲҲ layer-m faces} v(f)
```

This functional:

  В· vanishes on `imОҙВ№_m` вҖ” each layer-m edge appears in
    `(NS вҲ’ 1)(NT вҲ’ 1)` layer-m faces (even when both вүҘ 2);
  В· vanishes on cup-images of cocycles supported in OTHER
    layers (edge-disjoint property above);
  В· is NONZERO on layer-m face indicators outside cup-image of
    same-layer cocycles.

So the `c` independent `ПҲ_m`-functionals lift to `c` independent
2-cocycle classes in HВІ, each cup-trivial against its own
layer's cocycles.

вҹЁ0, hcвҹ©)` with the SAME edge_idx construction that appears in
the unfolded goal.  Fin proof terms are preserved exactly,

## Cup-image dual span (upper bound side)

The lower bound `codim вүҘ c` says the c ПҲ-discriminators witness
c independent classes in `HВІ_enr / cup-image`.  The matching
upper bound `codim вүӨ c` requires showing these c functionals
SPAN the dual of `HВІ_enr / cup-image`.  Captured parametrically
in `V33EnrichedParametricDualSpan.lean`.

### The dual-span scaffolding

Three structural pieces (all PURE):

  В· **ПҲ-linearity** (`psi_layer_linear`) вҖ” ПҲ_m distributes over
    pointwise XOR, lifted through the abstract `psiNatPos` fold.
  В· **Surjectivity** (`psi_layer_weighted_e_sum`) вҖ” the c-dim
    subspace `weighted_e_sum c b := ОЈ_m (b m) В· e_face_layer m`
    hits any target ПҲ-vector `b : Fin c вҶ’ Bool`.  So
    `ОЁ : EnrichedFaceVal c вҶ’ (Fin c вҶ’ Bool), v вҶҰ (m вҶҰ ПҲ_m v)` is
    surjective.
  В· **Canonical decomposition** (`psi_canonical_decomposition`)
    вҖ” every v = `weighted_e_sum c (m вҶҰ ПҲ_m v) вҠ• residual` with
    `ПҲ_m(residual) = false` for all m.

### Why PRIMARY (not FULL) cup-image

The cup-image must be against `cupOpp_param (starS i m) ОІ`
(S-star Г— anything) plus `cupOpp_param Оұ (incidT j m)` (anything
Г— T-incidence), NOT against arbitrary `cupOpp_param Оұ ОІ`.
The reason: by bilinearity, single-edge cups along face
diagonals reproduce every face indicator, so the FвӮӮ-span of full
cup-products = entire `EnrichedFaceVal c`.  Concrete witness
`psi_layer_arbitrary_cup_not_kill`: `ПҲ_0(e_0 вҲӘ e_4) = true` (the
diagonal pair (e_0, e_4) of face (0, 0, 0) cups to the single-face
indicator at (0, 0, 0), which ПҲ_0 detects).

Hence the inductive type `InPrimaryCupSpanPlusBoundary` admits
constructors only for ОҙВ№Пғ, starS-cup, incidT-cup, and XOR.

### Conditional codim upper bound + capstone

`codim_upper_bound_conditional`: IF the joint ПҲ-kernel is
contained in `InPrimaryCupSpanPlusBoundary`, THEN every v
decomposes canonically modulo `InPrimary` into its `weighted_e_sum`
representative.

`parametric_dual_span_capstone`: equivalently, the c
ПҲ-discriminators SPAN the dual of
`EnrichedFaceVal c / InPrimaryCupSpanPlusBoundary`.

### Cross-layer vanishing (structural piece of Direction B)

`starS_layer_disjoint` / `incidT_layer_disjoint`: at any layer
m' вү  m, the cocycle `starS i m` / `incidT j m` evaluates to
false on every layer-m' edge.  Proof via 9-block disjointness
(`nine_block_disjoint_op` in `NatBeqHelpers`): edge indices
`9В·a + r` for `a` the layer index and `r < 9` the in-layer offset
form disjoint blocks, so distinct layers give distinct indices.

Lifting: `cupOpp_{starS,incidT}_cross_layer_zero` (the cup
vanishes at every layer-m' face вҖ” the incidT case uses
`Bool.and_false` since the false factor sits on the right of
`&&`), then `psi_layer_{starCup,incidCup}_cross_layer` (the
XOR of 9 zeros).  All unconditional.

`primary_cup_span_soundness_on_layer`: combines the cross-layer
theorems with `primary_cup_span_soundness_conditional` to give
soundness requiring ONLY the on-layer kill (`m = m'` case)
as hypothesis.  All cross-layer cases are dispatched
automatically.

### Unconditional soundness at c = 1

`primary_cup_span_soundness_c1`: at c = 1 the on-layer
hypothesis collapses since `Fin 1` forces `m = вҹЁ0, _вҹ©`, and the
existing bottom-layer kills suffice.  Gives unconditional
`InPrimary вҠҶ joint ПҲ-kernel` at single-layer K_{3,3}.  The
remaining open piece for c = 1 unconditional `codim вүӨ 1` is
per-layer completeness.

## K_{3,3} вҶ” MГ¶bius P bridges

K_{3,3}^{(c=2)} differs from K_{3,2}^{(c=2)} in landing OFF the
canonical Pseq seedZero orbit (3, 3 lies on the diagonal
`{(N, N)}`, not on either thin chain).  Three bridges expose
the MГ¶bius P structure that survives the off-orbit position.

### Vertex state class вҖ” NS-scaled depth-1

`Mobius213K33StateClass.state_class_NSscaled_pell_capstone`
(PURE): K_{3,3}^{(c=2)}'s all-true vertex cochain projects to
state class `(3, 3) = NS В· Pseq seedZero 1`.  Iterating P:
`Pstep^n (3, 3) = NS В· Pseq seedZero (n+1)`.  Both components
satisfy the Pell-Fibonacci recurrence `a(n+2) + a(n) = NS В· a(n+1)`
on the NS-scaled orbit.

Unlike K_{3,2} (whose state lies directly on the orbit), K_{3,3}
lives on the `NS`-scaled diagonal вҖ” the symmetric subspace
invariant under the swap involution.  Under `Pstep`, the
diagonal goes off-diagonal: `Pstep(N, N) = (3N, 2N)`, hitting
the symmetric direction `(3, 2)`-style scaled by N.

### Edge multiplicity saturation at c = 3

`Mobius213K33c3StateClass.multCount_master` (PURE):
`multCount allTrueE = (9, 9, 9)`.  At c = 3 the three
multiplicity layers each carry exactly `NS В· NT = 9` cross-pair
edges.  The uniform `(9, 9, 9)` signature is the edge-level
mirror of the `(3, 3)` vertex state class вҖ” the all-true edge
cochain saturates each layer's 9-slot exactly.

### Numerical signature bridge

`Mobius213K33Bridge.k33_signature`: `6 = 2 В· NS`, `9 = NS В· NS`
вҖ” the vertex count is `NT_d В· NS` (with `NT_d = 2`, the
diagonal partition factor) and the cross-pair count is `NSВІ`.
Connects the cohomology-side counts to MГ¶bius P invariants
directly.

### K_{3,3} unified master

`K33Unified.k33_unified_master` (PURE) bundles all four tracks:

  В· (a) parametric вҲҖc codim вүҘ c (`V33EnrichedParametric`)

K_{4,3} is the first asymmetric (NS вү  NT) instance of the
classification, sitting at the Stern-Brocot mediant of (1, 1)
and (3, 2) вҖ” interior to the tree, not on either thin chain.

  В· 7 vertices (4 + 3)

{(0,1), (0,2), (0,3), (1,2), (1,3), (2,3)}`, the 3 T-pair

## Cross-graph universality of the S-row pattern

`CrossGraphPattern.cross_graph_S_row_dependence_pattern` (PURE)
bundles the K_{3,3} and K_{4,3} S-row dependences as instances of
ONE universal fact: at NT = 3, the 3 T-pair 4-cycles at any
S-pair sum to zero in FвӮӮ.

NS does NOT enter the proof вҖ” only NT = 3 and the specific S-pair
structure matter.  The same proof template ports across
bipartite graphs sharing NT = 3.  K_{3,3} (NS = 3) gives 3
S-pair instances; K_{4,3} (NS = 4) gives 6.  General K_{NS, 3}
gives `NS.choose 2`.

This cross-graph reusability is the operational content of the
Stern-Brocot classification's "axis decoupling": cohomological
proofs depending only on NT = 3 transport along every NT-fibre
of the Stern-Brocot tree.

## Mediant cohomology functor (count level)

The Stern-Brocot mediant `(NSвӮҒ, NTвӮҒ) вҠ• (NSвӮӮ, NTвӮӮ) = (NSвӮҒ+NSвӮӮ,
NTвӮҒ+NTвӮӮ)` lifts to a **Vandermonde decomposition** of every
cell count.  `MediantCohomologyFunctor.lean` (22 PURE, вҲ…-axiom):

  В· **Vandermonde-2 core**: `binom (a + b) 2 = binom a 2 +
    binom b 2 + a В· b` (`binom_add_2`).  The combinatorial
    heart вҖ” S-pairs of K_{a+b, *} split into intra-a, intra-b,
    and cross-ab pairs.
  В· **Vertex 2-term additive**:
    `V(a+c, b+d) = V(a, b) + V(c, d)` (`vertexCount_mediant`).
  В· **Edge 4-term Vandermonde**:
    `E^m(a+c, b+d) = E^m(a, b) + E^m(a, d) + E^m(c, b) +
    E^m(c, d)` (`edgeCount_mediant`).  Four edge classes:
    inner-1, cross-12, cross-21, inner-2.
  В· **Face factored VandermondeВІ**:
    `F(a+c, b+d) = (binom a 2 + binom c 2 + aВ·c) В·
    (binom b 2 + binom d 2 + bВ·d)` (`faceCount_mediant_factored`).
    9 terms total, each pairing one of three S-pair sources
    (intra-a / intra-c / cross-ac) with one of three T-pair
    sources (intra-b / intra-d / cross-bd).

Concrete marquee instance, K_{4,3} = K_{1,1} вҠ• K_{3,2} at
c = 2:

  В· `7 = 2 + 5` вҖ” vertex additivity
  В· `24 = 2 + 4 + 6 + 12` вҖ” edge 4-term
  В· `18 = (0 + 3 + 3) В· (0 + 1 + 2) = 6 В· 3 = 18`
    вҖ” face factored VandermondeВІ (4 of 9 terms non-zero;
    `binom 1 2 = 0` zeros out the remaining 5)

Anchors:
`K43_{vertex,edge,face}_from_mediant`,
`K43_face_9term_evaluation`,
`countTriple_mediant_decomposition` (3-component algebra law),
`mediant_cohomology_functor_capstone` (7-conjunct master).

The functor structure: assigning
`(NS, NT) вҶҰ (vertexCount, edgeCount, faceCount)` factors
through the Stern-Brocot mediant inductive structure.
Since every coprime `(NS, NT) вүҘ (1, 0)` is Stern-Brocot
reachable (`Mobius213SternBrocot.reachable_of_pos`), every
`K_{NS, NT}^{(c)}` count is computable as a finite sum of
products of `binom` values along the Stern-Brocot path from
seeds to (NS, NT).

`theory/essays/stern_brocot_as_universal_lattice.md` discusses
the structural significance.  The lift from cell-count
Vandermonde to actual cochain-space / Massey-class
decomposition remains open.
