# The c-counter as a layer count

The multiplicity `c` of `K_{NS, NT}^{(c)}` is not a depth parameter
of any fixed cohomology complex; it is the *layer count* of `c`
disjoint copies of the simple complex glued at the vertex level.
The cup-image codim in the enriched 2-complex grows linearly:
`codim ≥ c`, one independent Massey 5th-dim direction per layer.

## 213-native answer

A bipartite multigraph `K_{NS, NT}^{(c)}` is generated by
`(NS, NT, c)`.  The first two index the graph shape (vertex
partition + cross-pair structure); the third indexes the EDGE
multiplicity per cross-pair.  Standard cohomology of the
2-skeleton (simple 4-cycle faces using only mult-0 edges) makes
`c` invisible — `dim H² = 5` is independent of c (proved at
c = 2 via `V33` and c = 3 via `V33c3.c3_conjecture_refined`).

The c-counter lives ONE LEVEL DEEPER, in the enriched 2-complex
that admits mult-m face cycles for `m ∈ {0, …, c−1}`.  There it
manifests as `c` independent ψ-discriminators
(`V33EnrichedParametric.parametric_c_independent_h2_classes`).

## Derivation

The structural fact at the heart of the reframing: the c
multiplicity layers in `K_{NS, NT}^{(c)}` have DISJOINT edge sets.

```
edges at mult m = { c·(NT·i + j) + m : i ∈ Fin NS, j ∈ Fin NT }
                = { c·k + m : k ∈ Fin (NS·NT) }
```

For distinct `m, m' ∈ Fin c`, these sets don't overlap.  Hence
the enriched chain complex SPLITS:

```
C¹_enr = ⊕_{m ∈ Fin c} C¹_simple_m
C²_enr = ⊕_{m ∈ Fin c} C²_simple_m
δ¹_enr = ⊕_m δ¹_simple_m
```

This is a CATEGORICAL DIRECT SUM (see companion essay
`disjoint_layers_as_direct_sum.md`).  Cohomology distributes
through the direct sum:

```
H²_enr        = ⊕_m H²_simple_m
cup-image_enr = ⊕_m cup-image_simple_m
codim_enr     = c · (per-layer codim)
```

For K_{3,3} the per-layer codim is 1.  Hence total codim is c.
The `(c−1)` form is off-by-one; the correct parametrisation is
`codim ≥ c`.

The ψ-discriminator `ψ_m := XOR_{f ∈ layer-m faces} v(f)`
realises this concretely.  It vanishes on `imδ¹_m` (each
layer-m edge appears in `(NS − 1)(NT − 1)` layer-m faces — even
when both ≥ 2) and on cup-images from OTHER layers
(edge-disjointness).  The single-face indicator `e_face_(9m)`
has Kronecker-δ signature against the c discriminators
`(δ_{m,m'})_{m' ∈ Fin c}`, giving c independent non-coboundary
H²-classes.

## Dual function

This is not "the conjecture was wrong, fixed it to `c`".  The
conjecture posed the question in the wrong category: it asked
where in the SIMPLE complex `c` shows up.  Under the simple
complex `c` doesn't show up at all (codim stays 1, depth stays
4 — both invariant of c).  Reading the question as "in what
category does c become visible" forces the move to the enriched
complex, where the answer becomes a counting fact (number of
disjoint multiplicity layers) rather than a calculated invariant.

The classical reading would package the layer count as a
filtration depth, asking how *deep* a Massey product needs to be
to span the missing dimensions.  The 213 reading keeps the
Massey depth fixed at 4 (independent of c — see
`V33Massey4Fold` at c = 2 plus `psi_layer_rep4_eq_true_c{2..12}`
for c = 2..12) and exposes c as a *parallel* parameter: c
Massey witnesses, one per layer, all at the same depth.

## Cross-frame connections

The reframing connects four distinct readings of "c":

  · **Categorical**: c = number of summands in the direct-sum
    decomposition of the enriched chain complex
    (`V33EnrichedParametric` parametric formulation).
  · **Combinatorial**: c = number of disjoint edge layers in
    `K_{NS, NT}^{(c)}` (definition).
  · **Cohomological**: c = number of independent ψ-discriminators
    in H²_enr / cup-image
    (`parametric_c_independent_h2_classes`).
  · **Algebraic**: c = number of independent 4-fold Massey
    witnesses (`eta_ab_layer m`, `eta_cd_layer m` parameterised
    over `m ∈ Fin c`, each hitting its own ψ_m).

All four readings are the same number for the same structural
reason — the disjoint-layer factorisation forces the count to
be the same at every level.  This is the unifying content of
`research-notes/archive/G145_c_counter_structural_theory.md`
§Insight 1-2.

## Stern-Brocot decoupling

The (NS, NT) graph shape and the c multiplicity occupy ORTHOGONAL
axes.  K_{3,2}, K_{4,3}, K_{5,3}, K_{3,3} all sit at distinct
Stern-Brocot positions
(`BipartiteStermBrocotClassification.lean`); the c-counter
`codim ≥ c` proof transports along every NT-fibre without
recomputation (NS doesn't enter
`parametric_c_independent_h2_classes`).

This decoupling is the operational content of "c is a
structural parameter, not a calculated invariant" — calculation
would tie c to (NS, NT); structure makes them independent.
