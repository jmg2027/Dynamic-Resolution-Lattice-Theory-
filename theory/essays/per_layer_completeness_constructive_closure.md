# Layer-promotion as the structural lift — 8 explicit generators that close `codim ≤ c` at every Stern-Brocot position

The HARD direction `joint ψ-kernel ⊆ InPrimary` at every `c ≥ 1`
closes the `codim = c` upper bound unconditionally.  Its proof is
not an abstract dimension count but a **constructive
decomposition**: eight specific primary cup-products at single-
layer K_{3,3} span the dim-8 ψ-kernel, and layer-promotion
translates each generator to every multiplicity index `m ∈ Fin c`,
turning the per-layer fact into an ∀c theorem.  The promotion is
syntactically what direct-sum already said semantically — the
two readings collapse on the proof.

## 213-native answer

For `c = 1`, the 9-element face space `EnrichedFaceVal 1` of
`K_{3,3}` modulo the joint ψ-kernel is dim 1, generated by
`e_face_layer 1 ⟨0,_⟩`.  Equivalently: the joint ψ-kernel is
dim 8, generated by

```
g_1 = cupOpp(starS 0 0, e_edge (1, 0))   = e_{(0,0)} ⊕ e_{(0,1)}
g_2 = cupOpp(starS 0 0, e_edge (1, 1))   = e_{(0,0)} ⊕ e_{(0,2)}
g_3 = cupOpp(starS 0 0, e_edge (2, 0))   = e_{(1,0)} ⊕ e_{(1,1)}
g_4 = cupOpp(starS 0 0, e_edge (2, 1))   = e_{(1,0)} ⊕ e_{(1,2)}
g_5 = cupOpp(starS 1 0, e_edge (2, 0))   = e_{(2,0)} ⊕ e_{(2,1)}
g_6 = cupOpp(starS 1 0, e_edge (2, 1))   = e_{(2,0)} ⊕ e_{(2,2)}
g_7 = cupOpp(e_edge (0, 1), incidT 0 0)  = e_{(0,0)} ⊕ e_{(1,0)}
g_8 = cupOpp(e_edge (2, 1), incidT 0 0)  = e_{(1,0)} ⊕ e_{(2,0)}
```

— six row pair-differences (each face row contributes two), two
column pair-differences with linearly-independent row-sum
signatures.  Any face cochain `v` with `ψ_0(v) = 0` is
`⊕ᵢ bᵢ · gᵢ` with `b₁ … b₆ = v` at the six free positions and

```
b₇ = v(0,0) ⊕ v(0,1) ⊕ v(0,2)
b₈ = v(2,0) ⊕ v(2,1) ⊕ v(2,2)
```

The position-(1, 0) equation `v(1, 0) = b₃ ⊕ b₄ ⊕ b₇ ⊕ b₈` is the
ψ-kernel constraint `XOR_{(s, t)} v(s, t) = 0` rewritten.

For `c > 1`, `promote_face c m : EnrichedFaceVal 1 →
EnrichedFaceVal c` lifts each generator to layer `m`:

```
g_i^{(c, m)} := cupOpp(starS c i_S m, promote_edge β)
            (or  cupOpp(promote_edge α, incidT c i_T m))
```

These layer-m promotes lie in `InPrimary c` by direct constructor
application (`starCup`, `incidCup`).  `xor_aggregate c v c` sums
the `c` layer-m candidate functions and reconstructs `v`
pointwise.  `joint_psi_kernel_subset_primary` is the resulting
∀c statement.

## Derivation

The 8 generators come from a two-pass count.  First, the
ψ-kernel at single layer has dim `9 - 1 = 8` (one constraint from
ψ_0: XOR-sum vanishes).  Second, single-edge coboundaries
`δ¹(e_edge (i, j))` span only dim 4 (the cycle space of K_{3,3} is
9 - 6 + 1 = 4).  The remaining 4 dimensions must come from
primary cup-products.

`cupOpp(starS i 0, β)` parametrically depends on β over the 9
edge cochain values.  At face position (s, t), the cup formula
becomes `β(p_hi t) ⊕ β(p_lo t)`-style sum of single-edge values —
when β is a single-edge indicator `e_edge (k, l)`, the cup is
either zero (no matching edge in the face's diagonal pair) or a
face pair-difference `e_face(s, t) ⊕ e_face(s', t')`.  Choosing
β = `e_edge (1, 0), …, (2, 1)` gives `g_1, …, g_4` (row 0 and
row 1 pair-differences); `cupOpp(starS 1 0, ⋯)` gives `g_5, g_6`
(row 2).  The `incidT` cups give column pair-differences `g_7, g_8`
with row-sum signatures `(1, 1, 0)` and `(0, 1, 1)` — linearly
independent in F_2^3, completing the dim-8 span.

The decomposition `v = ⊕ᵢ bᵢ · gᵢ` is verified pointwise at all
9 face positions (`candidate_at_00`, `candidate_at_01`,
`candidate_at_02`, `candidate_at_10`, `candidate_at_11`,
`candidate_at_12`, `candidate_at_20`, `candidate_at_21`,
`candidate_at_22`).  Position (1, 0) is the only one using the
ψ-kernel hypothesis — exactly because (1, 0) is the position
*not* covered by any `b_i = v(i, j)` directly; the ψ-constraint
fills in its value as the XOR of the other 8.

## The `cong` constructor as funext-bypass

The candidate `⊕ᵢ bᵢ · gᵢ` agrees with `v` pointwise but is not a
function-literal equal to `v`.  `funext` would close this gap,
but `funext` uses `Quot.sound` — forbidden by the ∅-axiom
standard.  The bypass: add a constructor

```
| cong (v w : EnrichedFaceVal c) (h : ∀ s t m, v s t m = w s t m) :
    InPrimaryCupSpanPlusBoundary c w → InPrimaryCupSpanPlusBoundary c v
```

to the `InPrimary` inductive type.  Pointwise equality propagates
the InPrimary witness through the inductive structure itself —
the equivalence class is embedded *as a constructor*, not lifted
through an external axiom.  This is the 5th pattern in the
funext-avoidance family (`pure_funext_avoidance.md`), distinct
from Setoid Category because cong lives inside the inductive
predicate rather than outside it.

## Promotion preserves all 6 constructors

The lift `InPrimary 1 w → InPrimary c (promote_face c m w)` is by
induction on the 6 inductive constructors:

| Constructor | Promoted form | Realised via |
|---|---|---|
| `zero` | pointwise zero | `cong` to `InPrimary.zero` |
| `coboundary σ` | `δ¹(promote_edge σ)` | `cong` to `InPrimary.coboundary` |
| `starCup i ⟨0,_⟩ β` | `cupOpp(starS c i m, promote_edge β)` | `cong` to `InPrimary.starCup` |
| `incidCup j ⟨0,_⟩ α` | `cupOpp(promote_edge α, incidT c j m)` | `cong` to `InPrimary.incidCup` |
| `xor_add v w` | distributes pointwise | `cong` to `InPrimary.xor_add` |
| `cong v w h` | pointwise eq transports | `cong` (recursive) |

Each row's pointwise-equality lemma uses `promote_edge_at_layer_m`
(when `m' = m`) and the existing `cupOpp_{starS,incidT}_cross_layer_zero`
+ `starS_layer_disjoint` / `incidT_layer_disjoint` lemmas (when
`m' ≠ m`).  The cross-layer cases are exactly Direction A's
disjoint-layer factorisation (`theory/essays/disjoint_layers_as_direct_sum.md`)
re-used at the inductive lift.

`promote_edge` itself is defined as a 9-fold `cond` cascade
(rather than `Nat.div`/`Nat.mod` based lookup) — `Nat.div_eq_of_lt`
pulls `propext` via its `if`-branching, and an OR-cascade with
trailing `|| false` doesn't reduce by `rfl` for abstract Bool
tails.  The `cond` cascade reduces step-by-step:
`cond true x y = x` and `cond false x y = y` are both `rfl`,
chaining all the way to the unique matching offset.

## Dual function

Classical reading: the 8-dim ψ-kernel at K_{3,3} is computed by
constructing 8 explicit generators and verifying linear
independence; the c-fold lift is justified by direct-sum
decomposition of the enriched complex.  Two-step proof, each step
abstract.

213 reading: the 8 generators ARE primary cup-products of
specific cocycles with specific single-edge cochains — every
generator's existence is its construction.  Layer-promotion is
the syntactic shadow of layer-disjointness — the same
`9·m + r` block structure that closes Direction B's kill
(`nat_decide_add_left_assoc{1,2}` in `NatBeqHelpers`) also makes
`promote_edge` map each edge_idx to the right offset.  Same
equation, no separate justification step.

## Cross-frame

The closure appears in five readings:

  · **Combinatorial**: 8 face pair-differences span the
    XOR-sum-zero subspace of `Bool^9`; layer-disjointness gives
    `c · 8` total dimension at level c, matching `c · (9 - 1) = 8c`
    minus the c constraints from c ψ-discriminators = c.
  · **Categorical**: `EnrichedFaceVal c = ⊕_m EnrichedFaceVal 1`
    (one summand per layer); the promotion `promote_face c m`
    is the m-th coproduct inclusion; `xor_aggregate c v c` is
    the fold over the coproduct.
  · **Cohomological**: c ψ-discriminators span the dual of
    `EnrichedFaceVal c / InPrimary c` (the closed
    `parametric_dual_span_unconditional`); the 8 generators at
    each layer are an explicit basis for the cup-kernel
    component.
  · **Algebraic**: each generator is a primary cup-product of a
    `starS` or `incidT` cocycle with a single-edge cochain —
    the cup-product algebra applied 8 times per layer realises
    the kernel.
  · **Constructive (213-native)**: every face cochain in the
    joint ψ-kernel has an EXPLICIT 8-coefficient decomposition;
    every "kernel element" is a specific XOR of specific cup
    products, no abstract existence.

All five collapse onto the same set of primary cup products —
nothing is added when moving between readings, only the
labelling depth.

## Why the 9-element face is enough

A face space of size 9 admits a finite case analysis (2^9 = 512
cochains).  But the proof doesn't enumerate 512 cases: the 8
generators give a basis, and the decomposition `b_1 … b_8` is a
*function* of `v` that reads 8 values directly and assembles `b_7,
b_8` as 3-term XORs.  The 9-th value (position (1, 0)) is
constrained by ψ_0 = 0.

This is the 213 economy: the constraint `ψ_0 = 0` is read as
"v(1, 0) is determined by the other 8".  The position (1, 0) plays
no privileged role except that the 8 generators happen to expose
the other 8 positions directly.  A different choice of generators
would expose a different "constrained" position.  The asymmetry
is generator choice, not structural.
