# P-orbit closure master — 6-phase synthesis

Companion essay to the Lean modules formalising the P-orbit
closure programme.  Six phases assemble a structurally complete
closure theorem: the ring generated by `{L(k) = trace(P^k)}` and
atomic primes is the precise locus of every framework-natural
integer invariant, and atomic forcing extends from static shape
(PairForcing, Five) to dynamic orbit (OrbitForcing).

## Six phases

### Phase 1 — P self-reference via Cayley-Hamilton + Cassini

Module: `Mobius213/Px/CharPolySelf.lean` (11 PURE).

P's characteristic polynomial `χ_P(x) = x² − NS · x + det = x² − 3x + 1`
governs its trace orbit by Cayley-Hamilton:

  `L(k+2) = NS · L(k+1) − det · L(k)` — formalised as
  `L_cayley_hamilton`.

The Pell-Lucas discriminant `D = NS² − 4·det = 5 = d` and the
Cassini identity `L(n−1) · L(n+1) − L(n)² = d` (for `n ≥ 1`, with
`αβ = det = 1`) reconstruct atomic `d` from L-values alone:
`L(0) · L(2) − L(1)² = 2·7 − 9 = 5 = d`.

The atomic primes themselves seed the L-sequence:
`NT = L(0), NS = L(1)`.  Therefore the data needed to define P
(matrix entries, char-poly, discriminant) all lies in P's own
trace orbit.  P is **self-defining**: the dynamic orbit contains
the static defining data.

### Phase 2 — Inductive P-orbit ring

Module: `Mobius213/Px/POrbitRing.lean` (22 PURE).

`InPOrbitRing : Int → Prop` is the smallest predicate containing
the seeds `{NT, NS, d, 1, L(k) for k ≥ 0}` and closed under +, −, ·.
Bundled into a single Lean inductive.

The 12-prime catalog (mod-{2, 3, 5, 7, 11, 13, 17, 29, 37, 43, 53, 71})
is exhibited concretely as `p_orbit_ring_catalog_master`.

**Triviality consequence**: coprime seeds `(NT, NS) = (2, 3)` give
`1 ∈ ring` (Bezout: `1 = NS − NT`), hence the ring is all of ℤ.
*Membership is trivial*; the structurally meaningful invariant is
**P-orbit depth** — the minimum `K` for which the integer is
expressible using only `L(0), ..., L(K)` and atomic monomials.

### Phase 3 — `Theory.Atomicity.OrbitForcing` — recurrence is forced

Module: `Theory/Atomicity/OrbitForcing.lean` (13 PURE), the
8th file in the Atomicity cluster.

Given the atomic seeds `(NT, NS) = (2, 3)` and the canonical
Cayley-Hamilton target `L(2) = 7`, the 2nd-order linear recurrence

  `L(k+2) = a · L(k+1) − b · L(k)`,  `(a, b) ∈ {1, 2, 3}²`

has a **unique** coefficient pair: `(3, 1) = (NS, det)`.  Encoded
two ways: a Bool exhaustion (`boundedUniqueBool = true`) and 8
individual `non_canonical_fail_{i,j}` term-mode negations.

This lifts shape forcing to **orbit forcing**:

  · `PairForcing` → static pair `(2, 3)` unique
  · `Five` → static discriminant `d = 5` unique
  · `OrbitForcing` → dynamic recurrence `(3, 1)` unique

P's char-poly — and therefore the entire trace orbit — is fixed
by the atomic data.  P-iteration is not an independent choice.

### Phase 5 — Bipartite × Tripartite × P-orbit cross-product axes

Module: `Mobius213/Px/CrossProductAxes.lean` (17 PURE).

The triple-axis address structure `CrossAddress :=
(bipartiteCount, tripartiteCount, pOrbitDepth)`.  Atomic primes
seed minimal addresses:

  · NT → `(2, 2, 0)`
  · NS → `(3, 3, 1)`
  · d  → `(0, 5, 1)` (additive on P-axis: `L(1) + L(0) = d`)

Cross-link theorems:
  `bipartite_tripartite_cross`: `NS · NT = NT · 1 · NS = 6`
  `additive_sum_cross`: `NS + NT = d = 5`

Catalogued species (mod-13, mod-17, mod-29, mod-73) have explicit
addresses tracking each axis independently.  The 213 naturalness
boundary is this cross-product lattice.
