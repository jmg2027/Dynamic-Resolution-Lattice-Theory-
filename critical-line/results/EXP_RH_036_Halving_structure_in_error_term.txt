=== Test 3: WHY half? ===
  Combinatorial argument for the 1/2 exponent

  A closed NB walk of length n:
  v_0 -> v_1 -> ... -> v_n = v_0

  'Progressive' walks: visit n distinct vertices
    -> contribute to main term q^n

  'Returning' walks: revisit some vertices
    -> each revisit creates a PAIR (fwd + back)
    -> each pair consumes 2 steps without progress
    -> k pairs reduce effective length to n - 2k
    -> maximum k = n/2 (all paired)
    -> minimum effective length = n/2
    -> error contribution ~ q^{n/2}

  The '2' in 'pair of 2 steps' is:
    = dim_R(C) = 2 (forward + backward = ℝ²)
    = the doubly irreducible number
    = the reason 1/2 appears, NOT 1/3 or 1/4

  Verification on K_8:
  If all error comes from paired steps:
