# G183 — non-holonomicity as the residue's escape, and the practical edge of "pointing, not number"

**Tier 1 (volatile, synthesis).**  A deeper re-reading of the π / pointing / non-holonomicity
arc (`G170`, `G173`–`G176`) **after merging `main`**, which brought the foundational
state-machine reading of the residue (`theory/essays/the_residue_as_state_machine.md`), the
difference-Lens account of `ℤ` (`integers_as_difference_lens.md`), and the Newton–Gregory /
finite-depth-ring machinery (`Cauchy/{NewtonGregory, FiniteDepthAlgebra, BinomialTransform}`).
Two payoffs: a **unification** (non-holonomicity is the residue's source-without-enclosure at
the continued-fraction scale) and a **practical edge** (how π is actually computed, and a
ready test for "no recurrence exists").

## I. The unification — π's non-holonomicity *is* the µF/νF residue gap, one scale up

`the_residue_as_state_machine` proves the residue's foundational shape as an automaton: the
free-running counter trace `spineL` is a genuine behaviour (νF, `AntiRefl spineL`) that is **the
trace of no finite reachable state** (`CoResidue.spineL_escapes`,
`FlatOntologyClosure.object1_not_surjective`).  A *finite-state machine's* output is exactly a
sequence generated with **bounded memory** — and over `ℤ` that is exactly a **finite linear
recurrence** (holonomic / P-recursive: the next term from a finite window).  So:

> **holonomic = generated by a finite-state machine (bounded memory) ; non-holonomic = a trace
> that escapes every finite-state machine.**

Therefore the marathon's two ∅-axiom certificates are the *continued-fraction-scale* instances
of `spineL_escapes`:

  - `Cauchy/NonHolonomicWitness.superFact_nonHolonomic` — `(n!)ⁿ` escapes by **growth** (its
    trace outruns any bounded-memory machine's envelope, Klazar);
  - `Cauchy/ZeroRunNonHolonomicWitness.chi_nonHolonomic` — the powers-of-two indicator escapes
    by **sparseness** (a bounded `0/1` trace whose zero-runs defeat any homogeneous finite
    recurrence — the `counter_no_return` "never settles" face, in the sparse direction).

π's regular continued fraction is *conjectured* to escape the same way — its partial quotients
are the free-running, non-recurring counter (Gauss–Kuzmin), the canonical residue trace.  So
**"π's CF is non-holonomic" = "π's CF trace is enclosed by no finite-state machine" = the
residue's source-without-enclosure, read at the CF scale.**  The foundational `spineL_escapes`
and the CF-scale `chi_nonHolonomic` are *the same theorem at two cofinal ends*.

**The FSM reading is now itself a theorem, not only an essay.**  The classic finite-state
machine is the **autonomous** (time-invariant) finite-window rule `AutoRec`
(`Cauchy/ZeroRunNonHolonomic`); `two_continuations_not_autoRec` is the Myhill–Nerode kernel — a
pattern with two distinct continuations is generated by no such machine — and
`ZeroRunNonHolonomicWitness.chi_not_autoRec` proves the powers-of-two trace escapes *every*
autonomous finite-state machine (an all-zero window is continued by both `0`, mid-run, and `1`,
just before a power of two).  So `χ` is now non-holonomic in **two complementary senses**:
`¬ HomogRec χ` (no *time-varying* homogeneous P-recursive recurrence) and `¬ AutoRec χ` (no
*time-invariant* finite-state machine) — the two machine classes whose union the residue
escapes.

## II. The difference-Lens ring — what "tier" measures

`integers_as_difference_lens`: `ℤ` is the readout group in which `Δ s n = s(n+1) − s n` closes
under iteration.  `Cauchy/FiniteDepthAlgebra`: the **finite-Δ-depth** `ℤ`-sequences form a
**ring** (closed under `+`, `·`, shift; depths add under product, `polyDepthZ_mul`), and
`Cauchy/NewtonGregory` gives their exact normal form (`reconstruct`: a depth-`d` sequence *is*
`Σ_{i≤d}(Δⁱs 0)·binom(·,i)`) and growth bound (`poly_bound`, which **closed Open Problem #4** —
the `QuasiPolyCF ⟹ polynomially-bounded ⟹ μ=2` bridge, `Cauchy/QuasiPolyBound`).

So the CF-holonomicity **tier** is a stratification *by where a sequence sits relative to this
ring*: tier 0/1 (periodic/Hurwitzian) live **inside** the finite-depth ring (per residue
class); the genuine non-holonomic tier is **outside it** — no finite-depth (no finite generating
machine) describes the partial-quotient sequence.  "Holonomicity is a property of the pointing,
not the number" (`PresentationDependence`, `theory/.../phi_pi_poles.md`) reads, in this frame,
as: **each pointing is a choice of state-machine for the same residue; the residue is the
behaviour all of them converge to and none generates as a finite state.**

## III. The practical edge

The non-holonomic/holonomic line is the line between *has a finite generating machine* and
*doesn't* — and that line is where real computation lives.

**1. Computing π fast = choosing a *holonomic* pointing.**  This is the working principle of
arbitrary-precision computation, and it is *exactly* "holonomicity is a property of the
pointing" in practice:

  - Record π computations (Chudnovsky brothers; y-cruncher) use the **Chudnovsky hypergeometric
    series** — a *holonomic* (P-recursive-term) pointing — because holonomic series admit
    **binary splitting** and run in quasi-linear time.  π's **continued fraction is
    non-holonomic** and is *useless* for fast computation.  Same number, two pointings; you
    compute through the holonomic one.  The CF being non-holonomic is not an obstacle to *having*
    the digits — it just means *that* pointing isn't the computational one.
  - The general engine is **creative telescoping / the holonomic systems approach** (Zeilberger;
    Zeilberger's algorithm): holonomic objects have *finite descriptions* a computer algebra
    system can store, sum, and certify; non-holonomic ones do not.  The marathon's distinction
    is the dividing line of that entire field.
  - **BBP**-type formulas (spigot / `n`-th-digit-without-predecessors) are a further holonomic
    pointing, practical for digit spot-checks.

**2. Sequence generation, LFSRs, compression.**  A **C-finite** sequence (constant-coefficient
recurrence) over `GF(2)` is exactly a **linear-feedback shift register** output (CRC, stream
ciphers, PRNGs) — finite-state, bounded memory.  The marathon's non-holonomic witnesses are the
**negative** side: the powers-of-two indicator and `(n!)ⁿ` admit **no** finite linear recurrence
— no LFSR, no closed recurrence, not compressible to "finite rule + finite seed."  So the
**zero-run criterion is a usable test**: *a `0/1` sequence with arbitrarily long zero-runs and
infinitely many ones has no finite homogeneous linear recurrence* (`zero_run_not_homogRec`) —
i.e. in OEIS / automated-conjecture / "guess-the-recurrence" practice, **stop looking for a
linear recurrence**; one provably cannot exist.

**3. Newton–Gregory interpolation = classical numerical analysis.**  `Cauchy/NewtonGregory`
formalizes the exact forward-difference calculus (`s(m+n)=Σ binom(n,j)(Δʲs)(m)`, the
reconstruction, the growth bound) — the basis of **difference tables**, **Newton
forward-difference interpolation**, and **Newton–Cotes** numerical integration.  A data sequence
is degree-`d` polynomial-interpolatable *iff* its difference table bottoms out at depth `d`
(finite-depth ring) — directly the holonomicity test on the practical, table-of-values side.

## IV. The one-paragraph synthesis

A real is a residue: reached by no pointing, pointed at by all (`object1_not_surjective`).  A
*pointing* is a state machine that emits rational approximants; **holonomic** pointings are the
finite-state (bounded-memory, finite-recurrence) ones, and they are the **computable, fast,
compressible** ones.  π's continued fraction is (conjecturally) a free-running counter that no
finite-state machine encloses — the residue's source-without-enclosure at the CF scale, the
foundational `spineL_escapes` one level up.  We still get `3.141592…` because π admits **other**,
holonomic pointings (Chudnovsky, BBP), and *that* — compute through the holonomic presentation,
never the non-holonomic one — is the practical heart of how π, and every constant, is actually
computed.
