## 1. 왜 이 분야인가

분석학 213 이 100% 마라톤 했으나, 213 *자체* 의 deepening 은
별도 작업 영역.  특히:

- **Lens framework** 의 더 깊은 형식화
- **Raw 공리 4 clause** 의 모든 alternative axiom 시도 + 모두
  collapse 증명 (이미 일부 있음 → 확장)
- **Falsifiability §5.2.1** 의 *기계적 감사관*
- **Bridge to ZFC** — 213 → ZFC translation 명시
- **Bridge from ZFC** — ZFC → 213 reduction (가능한 부분)

이게 *무한 마라톤*.  매번 분야 마라톤 종료 시 213 자체 도구도
한 단계 deepen.

## 2. 213-native 등장

### 2.1 Lens classification

Lens 가 functor 가 아닌 (`19_lens_not_functor.md`).  그렇다면
*무엇* 인가?  Lens 의 carrier:
- output type
- distinguishability preservation
- combine 대칭성

체계적 classification 필요.

### 2.2 Raw 공리 alternative

`AxiomMinimality.lean` 이 4 clause 검증.  더 *대안* 시도:
- 3-clause? (모두 collapse)
- 5-clause? (redundant)
- distinctness 약화? (specific case)
- 다른 atomic 수? (d=5 만 자연, 다른 d 시도)

각 alternative 시도 + Lean 검증.

### 2.3 Falsifiability 기계 감사관

매 마라톤 종료 시:
- `#print axioms` 검증
- *어떤* sorry 도 없는지 확인
- *어떤* external import (Mathlib, Classical) 도 없는지

자동 스크립트 + CI 통합.

### 2.4 Bridge to other foundations

213 → ZFC: Cut 을 Cauchy class 로 mapping.
213 → HoTT: dyadic tree 의 univalence?
213 → Type theory: dependent type 의 213-form?

각 bridge 가 *213 의 위상* 명시.

### 2.5 Cohomology framework 일반화

분석학 213 의 1-cochain → general n-cochain 자연 lift.
Simplicial cohomology (Cech, de Rham) 의 213-native form.

### 2.6 Categorical structure (조심!)

CLAUDE.md: "Lens ≠ functor".  Category 도입 시 *조심*.
하지만 213 위에서 *어떤* 적절한 구조 (Lens-cat?) 가능 여부
탐구.
